# OpenPets v3.1 Release Notes

## Why v3.1 exists

OpenPets v3.0 shipped the first SDK v3 plugin release, but the production plugin
catalog and desktop installer path exposed release-blocking issues:

- catalog cards could show unresolved `$t:plugin.name` / `$t:plugin.description`;
- catalog ZIPs with plugin `locales/*.json` were rejected by the released desktop
  ZIP validator as undeclared files;
- installed/local broken plugin records could get stuck when their install folder
  was already missing;
- catalog cards showed generic host outline icons instead of the plugin's bundled
  emoji-style SVG icon before install.

v3.1 is the clean forward fix. Do not optimize new plugin/catalog behavior around
v3.0 compatibility. Users on v3.0 should update/reinstall to v3.1.

## Plugin platform fixes

- Desktop accepts proper SDK v3 catalog ZIPs with bundled `locales/<locale>.json`.
- Catalog and ZIP manifest metadata must match using resolved user-facing strings,
  not `$t:` refs.
- Catalog entries can include `iconDataUrl` so Control Center catalog cards render
  the same bundled emoji-style SVG icons users see after install.
- Stale local-dev plugin records can be uninstalled when the expected
  `plugins-dev/<id>` directory is already missing, while unsafe paths/symlinks are
  still rejected.
- Plugin release validation now exists at the root:
  - `pnpm plugins:validate-release`
  - `pnpm plugins:validate-live`

## Release order

Use this order for v3.1 and later plugin-bearing desktop releases:

1. Land desktop/plugin compatibility changes.
2. Bump desktop/root version for the new release.
3. Run:
   ```bash
   pnpm plugins:package
   pnpm plugins:validate-release
   pnpm --filter @open-pets/desktop test
   ```
4. Build and publish the desktop release.
5. Publish plugin ZIPs and deploy the web catalog:
   ```bash
   pnpm plugins:publish
   cd web && ./deploy.sh
   ```
6. After deploy completes, verify production:
   ```bash
   pnpm plugins:validate-live
   ```

## Important compatibility note

The proper v3.1 plugin catalog should use `minOpenPetsVersion: "3.1.0"` for the
SDK v3 plugin lineup. This prevents v3.0 from installing packages it cannot
validate correctly.

Do not remove `locales/*.json` from v3 plugin ZIPs as a long-term workaround.
Locales are part of the v3 plugin package contract.
